This page displays configuration of the review checkers.

The Stork has a built-in feature to review the daemon configuration. It detects misconfiguration or popular mistakes and produces suggestions on improving performance or maintainability. The review report is created using the configuration checkers. Each checker is responsible for detecting a single issue.

Sometimes, the issue reported by the checker is expected, and the administrator decides to keep the configuration as-is. In this case, it may be helpful to disable a particular configuration checker to avoid creating unnecessary reports.

You can use this page to disable some checkers globally for every Kea daemon. This configuration may be overwritten for a specific daemon on its daemon page under the configuration review section.
